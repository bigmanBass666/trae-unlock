# Tasks: 跨会话 Agent 动态规则遵守系统

## 任务概览

本系统采用**渐进式实现策略**：先在 trae-unlock 项目中验证核心机制，再抽象为可移植框架。

---

## Phase 1: 基础设施搭建（规则仓库 + 规则定义）

- [ ] **Task 1.1**: 创建 `rules/` 目录并定义 4 个核心规则文件
  - [ ] 1.1.1 创建 `rules/core.yaml` — 核心必读规则（文档阅读要求、新发现记录规范）
  - [ ] 1.1.2 创建 `rules/workflow.yaml` — 工作流程规则（探索→修改→测试→提交循环）
  - [ ] 1.1.3 创建 `rules/git.yaml` — Git 提交规范（格式、强制 push）
  - [ ] 1.1.4 创建 `rules/safety.yaml` — 安全规则（密钥保护、.git 完整性）
  - **验证标准**：每个 YAML 文件符合 spec 中定义的 schema，包含 id/name/category/priority/enabled/description/actions/enforcement 字段

- [ ] **Task 1.2**: 从现有 AGENTS.md 提取规则并结构化
  - [ ] 1.2.1 将 AGENTS.md 中的"新会话开始前必读"章节 → 迁移到 `rules/core.yaml`
  - [ ] 1.2.2 将"核心规则"章节（更新文档要求）→ 迁移到 `rules/core.yaml`
  - [ ] 1.2.3 将"Git 提交规则"章节 → 迁移到 `rules/git.yaml`
  - [ ] 1.2.4 将"安全建议"相关内容 → 迁移到 `rules/safety.yaml`
  - **验证标准**：所有现有规则都被完整保留，无信息丢失

---

## Phase 2: 规则引擎开发（核心脚本）

- [ ] **Task 2.1**: 实现 `scripts/rules-engine.ps1` 基础功能
  - [ ] 2.1.1 实现 YAML 解析器（使用 PowerShell 原生能力或简单正则，避免外部依赖）
  - [ ] 2.1.2 实现规则扫描逻辑（递归读取 rules/*.yaml）
  - [ ] 2.1.3 实现过滤逻辑（只加载 enabled: true 的规则）
  - [ ] 2.1.4 实现优先级排序（high > medium > low）
  - **验证标准**：执行脚本后输出格式化的 Markdown 规则文本

- [ ] **Task 2.2**: 增强 Markdown 输出质量
  - [ ] 2.2.1 按类别分组显示规则（core / workflow / git / safety）
  - [ ] 2.2.2 为每条规则生成清晰的标题和操作步骤
  - [ ] 2.2.3 添加统计信息（共 X 条规则，Y 条启用，Z 条禁用）
  - **验证标准**：输出文本可直接被 AI 理解和遵循

- [ ] **Task 2.3**: 实现 CLI 参数支持
  - [ ] 2.3.1 支持 `--check` 参数（语法验证模式）
  - [ ] 2.3.2 支持 `--list` 参数（列出规则状态摘要）
  - [ ] 2.3.3 支持 `--output` 参数（指定输出文件路径）
  - **验证标准**：`./rules-engine.ps1 --check` 能正确检测 YAML 语法错误

---

## Phase 3: AGENTS.md 重构为路由器

- [ ] **Task 3.1**: 重构 AGENTS.md 为轻量级入口
  - [ ] 3.1.1 移除所有硬编码的规则细节（从 ~128 行缩减至 ~30 行）
  - [ ] 3.1.2 编写强制性引导指令（⚠️ 必须先运行规则加载命令）
  - [ ] 3.1.3 添加规则引擎调用示例（powershell scripts/rules-engine.ps1）
  - [ ] 3.1.4 添加缓存/刷新说明（修改 rules/ 后重新生成）
  - **验证标准**：AGENTS.md 总行数 < 50 行，且不包含任何具体规则内容

- [ ] **Task 3.2**: 测试路由器机制有效性
  - [ ] 3.2.1 模拟新会话场景：AI 读取 AGENTS.md 后是否知道要运行规则引擎
  - [ ] 3.2.2 验证规则引擎输出是否能被正确嵌入上下文
  - **验证标准**：通过实际对话测试确认 AI 会主动执行规则加载命令
