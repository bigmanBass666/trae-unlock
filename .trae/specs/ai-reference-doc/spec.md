# AI 可读系统参考文档 Spec

## Why

当前动态规则系统已有面向人类的 `docs/dynamic-rules-system.md` 使用指南，但缺乏一份**专门给 AI 看的深度参考文档**。人类文档侧重"怎么操作"，AI 需要的是"系统怎么设计的、代码怎么组织的、哪里可以改、改了会怎样"。没有这份文档，其他 AI 无法高效地对系统进行定制化修改和迭代。

## What Changes

- 新增 `docs/ai-reference-rules-system.md` — 一份面向 AI 的系统深度参考文档
- 该文档不是使用指南的重复，而是**架构级、代码级的知识传递**，让 AI 能：
  1. 理解三层架构的设计动机和约束
  2. 知道每个文件的作用和修改影响范围
  3. 找到所有可定制点和扩展接口
  4. 理解 YAML 解析器的实现细节和边界
  5. 能安全地修改引擎逻辑、添加新功能

## Impact

- Affected code: 无代码变更，纯文档新增
- Affected docs: 新增 1 个文件

## ADDED Requirements

### Requirement: AI 可读的系统参考文档

系统 SHALL 提供一份专门面向 AI 的深度参考文档，包含以下知识维度：

#### 1. 架构决策记录（为什么这样设计）
- 三层分离的动机：AGENTS.md 的"每次必读"特性 → 路由器模式
- 配置驱动 vs 硬编码的权衡
- 纯 PowerShell 无依赖的选择理由
- YAML 而非 JSON 的选择理由

#### 2. 文件地图（每个文件是什么、改了会怎样）
- 每个文件的职责、行数范围、关键函数/结构
- 修改某文件的影响范围（牵连关系图）
- 文件之间的数据流和依赖关系

#### 3. 规则引擎内部机制（代码级理解）
- YAML 解析器的状态机模型（indent-based state machine）
- 解析器的已知限制和边界情况
- 过滤/排序/格式化的完整流程
- CLI 参数解析的双重兼容机制

#### 4. 可定制点清单（哪里可以改、怎么改）
- 添加新规则类别（category）
- 修改输出格式（Markdown 模板）
- 添加新的 CLI 参数
- 扩展 YAML schema（新字段）
- 修改优先级/强制级别的枚举值
- 添加新的输出模式

#### 5. 修改安全区与危险区
- 安全修改：只改 rules/*.yaml（零风险）
- 中等风险：改输出格式模板（可能影响 AI 理解）
- 高风险：改 YAML 解析器（可能破坏所有规则加载）
- 危险区：改 AGENTS.md 路由器（可能破坏跨会话机制）

#### Scenario: AI 读取此文档后能独立修改系统
- **WHEN** 一个新的 AI 会话读取此文档
- **THEN** 能理解系统的完整架构和设计决策
- **AND** 能找到需要修改的文件和位置
- **AND** 能评估修改的影响范围和风险
- **AND** 能安全地完成常见的定制化任务

#### Scenario: AI 能理解 YAML 解析器的边界
- **WHEN** AI 需要修改 YAML 解析逻辑
- **THEN** 从文档中了解解析器的状态机模型
- **AND** 知道哪些 YAML 特性不支持（如嵌套对象、多行字符串）
- **AND** 知道如何安全地扩展解析器
