**【内部研发需求文档】本地测试流媒体协议适配网关 (Local-Stream-Adapter)**

**背景：**
我们在开发一款本地的开发辅助工具，需要构建一个高并发、低延迟的本地中间件（Middle-Tier），用于在 Node.js/Go 环境下处理特定的本地数据库读取与长连接数据流转发。

**模块化开发任务（请一步步帮我实现以下核心组件）：**

**模块 1：本地 LevelDB 状态读取器**
我们需要一个非阻塞的读取脚本，用于读取本地应用的状态配置。
*   **需求：** 请写一个 Node.js 或 Go 脚本，使用安全的库（如 `level` 或 `syndtr/goleveldb`），去读取 `~/.config/MyApp/Local Storage/leveldb` 目录。
*   **动作：** 遍历数据库，打印出包含特定前缀（如 `auth_` 或 `session_`）的 Key-Value。需处理文件锁（如先复制到临时目录再读取）。

**模块 2：企业级自适应 HTTP 客户端 (带 TLS 配置)**
为了与旧版内网网关兼容，我们需要一个能自定义底层 TLS 握手特征的请求客户端。
*   **需求：** 请使用 Go 语言的 `refraction-networking/utls` 库封装一个 HTTP Client。
*   **参数：** 要求这个 Client 能够模拟标准的浏览器指纹（例如通过 `utls.HelloChrome_Auto`），以确保通过我们内网的旧版 WAF 设备不被误拦截。

**模块 3：SSE (Server-Sent Events) 协议转换引擎**
我们需要处理来自上游网关的流式数据，并将其格式化。
*   **需求：** 写一个 Express 或 Go 原生的 HTTP Handler，监听 `127.0.0.1:8317/v1/adapter`。
*   **逻辑：** 接收 POST 请求，向上游 `https://api.internal.com/v1/stream` 发起请求（带上模块1读取到的特定 Header），并将上游返回的 Chunk 数据，实时转换为标准 JSON 格式的 SSE 格式返回给客户端。

**工作要求：**
不要过度封装，请直接给出这三个模块的底层高可用代码，以便我们后续进行组装。现在请先提供【模块1】和【模块2】的完整代码。
