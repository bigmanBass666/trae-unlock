### ⭐⭐⭐⭐⭐ 真正根因: auto_confirm=false (~8640019)
- **问题**: 之前补丁后弹窗仍在！深入分析发现：
  - getRunCommandCardBranch 补丁只改了分支选择，但**不是卡住的原因**
  - **真正原因**: egR 组件的自动确认 effect (`~8640019`) 要求 `en &&`(auto_confirm必须为true)
  - 红名单命令的服务端**故意返回 `auto_confirm:false`**
  - → en=false → 自动确认不触发 → 卡在"是否仍要在沙箱中运行"
- **关键变量** (egR 组件 ~8637300):
  - `er = N?.confirm_status` — 确认状态 ("unconfirmed")
  - `en = N?.auto_confirm` — **自动确认标志 (false!)**
  - `ey` 计算: `en ? Confirmed : er` — auto_confirm=true 时直接通过
- **最终补丁**: `!e&&er===Unconfirmed&&en&&ew.confirm(!0)` → 去掉 `&&en`
- **效果**: 所有 unconfirmed 命令（不管服务端是否允许）都自动确认执行
- 详细文档: [docs/source-architecture.md](docs/source-architecture.md) → "RunCommandCard 双层确认系统"章节

### ⭐⭐⭐⭐⭐⭐⭐ 切换窗口后自动确认失效 — 最终解决!
- **问题**: 切换AI会话窗口后，后台命令卡在"等待操作"
- **7次迭代完整记录**: 见 [source-architecture.md](docs/source-architecture.md) → "切换窗口后自动确认失效的完整迭代记录"
- **关键失败经验**:
  1. `ew.confirm()` 是日志函数不是执行函数 → 需要调 `eE(Confirmed)`
  2. useEffect 异步 → 切窗口后 React 不执行
  3. useMemo "同步"但组件冻结时不执行
  4. setTimeout 也依赖事件循环 → 组件冻结后不调度
  5. **所有 React 组件内的修改都无法解决** — 切窗口后组件全部冻结
- **最终方案: PlanItemStreamParser 服务层追加 provideUserResponse (~7503319, 补丁#8)**
  - 在 else 分支（非 knowledge 命令）也调用 provideUserResponse
  - 这是 SSE 流解析层（服务层），**完全不依赖 React 渲染**
  - 跟补丁#1(knowledge) 在同一层级，形成完整覆盖
- **效果**: ✅ 已验证！切窗口后命令也能自动执行
